

在新建项目时有两个可选选项，一个是继承全局第三方库，另一个提供给所有项目；
勾选Inherit global site-packages（继承全局第三方库）可以使用base interpreter（基础或全局环境）中的第三方库，不选将和外界完全隔离；
勾选Make available to all projects（提供给所有项目）可将此虚拟环境提供给其他项目使用。
我猜想勾选前者的项目在添加第三方库时应该是添加在全局的。
Visual Studio那边添加的第三方库应该也是添加在全局的

以前的项目，在更新解释器版本时可能对原有项目解释器不太兼容，那么可以删除原项目的的解释库文件在剩余原有文件基础上新建项目。
原有的项目运行和打开都有一点问题
